# 08. 페이즈 & 상태 머신

게임 전체의 상위 흐름. 매 화면(Screen)이 어떤 상태에서 어떤 입력을 받아 어떤 상태로 가는지 명시.

## 최상위 상태

```typescript
type AppState =
  | { kind: 'title' }                              // 슬롯 선택 화면
  | { kind: 'slot'; slotIndex: number; slot: Slot } // 선택된 슬롯의 진행 상태

// Slot 내 상태는 SlotState (01 문서):
type SlotState =
  | { kind: 'empty' }                              // 빈 슬롯 (선택 시 새 캐릭터 생성)
  | { kind: 'atRest' }                             // 휴식처 메뉴
  | { kind: 'inStartPhase'; pendingSkillChoice?: SkillOfferContext }
  | { kind: 'inRun'; run: RunState }

// RunState 내:
//   currentEvent      - 이벤트 진행 중
//   currentCombat     - 전투 중
//   둘 다 없으면      - 맵 탐색 모드
```

## 화면 전이 다이어그램

```
┌────────────────┐
│ TitleScreen    │  슬롯 5개 + [종료]
└─┬──────────────┘
  │ 슬롯 선택
  ▼
┌────────────────────────────┐
│ SlotMenuScreen             │  슬롯 빈/비어있지 않음 분기
│                            │
│ [empty]                    │
│  → "새로 시작"              │
│    → NewCharacterScreen    │  이름 입력
│      → StartPhaseScreen    │
│                            │
│ [non-empty]                │
│  → "이어하기" / "새로 시작" │
│  새로 시작 선택 시 확인 후  │
│  슬롯 wipe + NewCharacter  │
└──┬─────────────────────────┘
   │ 이어하기 (slot.state에 따라 분기)
   │
   ├─[empty]──→ (위 새로 시작 흐름)
   │
   ├─[atRest]──→ RestHubScreen
   │
   ├─[inStartPhase]──→ StartPhaseScreen
   │
   └─[inRun]──→ RunScreen (route by currentEvent/currentCombat)
```

## TitleScreen

```
┌─ textcrawlergame ──────────────────────────┐
│                                            │
│  > 슬롯 1: (빈 슬롯)                       │
│    슬롯 2: 헬리오스 — Lv 7 (탐험 중)       │
│    슬롯 3: (빈 슬롯)                       │
│    슬롯 4: 카르나 — Lv 12 (휴식처)         │
│    슬롯 5: (빈 슬롯)                       │
│                                            │
│  [메타 골드: 1240G    인벤: 18/25]         │
│                                            │
│    설정                                     │
│    종료                                     │
└────────────────────────────────────────────┘
```

입력:
- 방향키 ↑/↓: 포커스 이동
- Enter: 선택 → SlotMenuScreen
- 우측 패널: 포커스된 슬롯의 통계 (탐험 시간, 사망 횟수, 진행 깊이)

## StartPhaseScreen

```
┌─ 시작 페이즈 ────────────────────────────────┐
│ 차원의 안내자가 묻는다:                       │
│ "스킬 상자를 구매하겠는가?"                   │
│                                              │
│ 보유 메타 골드: 1240G                        │
│                                              │
│ > 최하급 상자 구매 (50G)                     │
│   하급 상자 구매 (150G)                      │
│   중급 상자 구매 (400G)                      │
│   상급 상자 구매 (1000G)                     │
│   최상급 상자 구매 (회색, 2500G 필요)        │
│   구매하지 않고 출발                          │
└──────────────────────────────────────────────┘
```

비활성 항목 (골드 부족)은 회색 + 사유 표시.

우측 패널: 포커스된 상자의 등급별 풀 미리보기 (가능한 스킬 일부 + 가중치).

선택 후:
- 구매: `onBuySkillBox(grade)` → 즉시 랜덤 스킬 1개 부여 → "획득: {스킬명}" 알림 → 출발
- 미구매: 즉시 출발

신규 캐릭터 + 인벤에 카드 있음:
- 시작 페이즈가 끝나면 첫 노드(여정의 시작 이벤트)에서 "차원 창고에서 꺼내기" 선택지 활성

기존 캐릭터 (휴식처 출발):
- "회수 마크" 한 카드들이 자동으로 덱에 합류
- 여정의 시작 이벤트는 신규 캐릭터에게만 등장 — 휴식처 출발 시는 일반 시작 이벤트 또는 바로 맵

## RestHubScreen

자유 메뉴, 무한 머무를 수 있음:

```
┌─ 휴식처 ──────────────────────────────────────┐
│ 헬리오스 (Lv 7) — HP 42/70                    │
│ 메타 골드: 1240G   인벤: 18/25                │
├──────────────────────────────────────────────┤
│ > 이번 런 카드 보기 (12장 미정리)             │
│   인벤토리 관리                                │
│   인벤토리 용량 업그레이드 (다음: 30칸 / 250G)│
│   다음 차원문으로 출발 →                      │
│   타이틀로 돌아가기                            │
└──────────────────────────────────────────────┘
```

각 메뉴는 서브 화면:
- **이번 런 카드** → `LastRunCardsScreen` (보관/판매)
- **인벤토리 관리** → `InventoryScreen` (회수 마크/판매/조회)
- **용량 업그레이드** → `CapacityUpgradeScreen` (확인 모달)
- **다음 차원문** → 자동 폐기 확인 모달 → StartPhaseScreen
- **타이틀로** → 저장 후 TitleScreen

## RunScreen (라우터)

`slot.state.run` 의 하위 상태에 따라:

```typescript
function routeRunScreen(run: RunState): ReactNode {
  if (run.currentCombat) return <CombatScreen />;
  if (run.currentEvent)  return <EventScreen />;
  return <MapScreen />;
}
```

### MapScreen

```
┌─ 맵 ───────────────────┬─ 노드 상세 ──────────┐
│   x:0  1  2  3  4      │ 일반 전투             │
│ y0  ?  ? [R] ?  ?      │ 좌표: (2,2)           │
│ y1  ?  ?  ?  ?  ?      │ 적: 가시도적 1마리    │
│ y2  ? [E] *  ?  ?      │ 보상: 골드 + 카드     │
│ y3  ?  •  *  •  ?      │                       │
│ y4  ?  ? [S] ?  ?      │ [→] 이동              │
├────────────────────────┤                       │
│ 이동 가능:             │                       │
│ > 북: 일반 전투        │                       │
│   동: 상점             │                       │
│   서: ? 미지의 이벤트  │                       │
└────────────────────────┴───────────────────────┘
```

입력:
- 방향키: 이동 가능 목록 포커스
- Enter: 선택 노드로 이동
- M: 맵 전체 보기 (이미 위에 표시 중이면 생략)
- Esc: 휴식처로 돌아갈지 모달 (탐험 포기 — 디자인 결정 필요)

> **결정**: 탐험 도중 휴식처로 강제 복귀 가능? 디자인상 위험. v1 안 됨.

### CombatScreen

```
┌─ 손패 ──────────────┬─ 카드 상세 ────────────┐
│ > 타격 (1)          │ 타격                   │
│   수비 (1)          │ 비용: 1                │
│   강타 (2)+         │ 타입: 공격             │
│   망토와 단검 (1)   │                        │
│   회복의 빛 (1)     │ 적에게 6의 피해.       │
├─────────────────────┤                        │
│ 적 (← 방향키)       │ [Enter] 사용           │
│ > 노예상인 HP 46/46 │ [Esc]   취소           │
│   의도: 공격 12     │                        │
├─────────────────────┼────────────────────────┤
│ HP 70/72  방어 0    │ 덱 15  버림 5  소멸 0  │
│ 에너지 3/3          │                        │
│                     │ [E] 턴 종료            │
└─────────────────────┴────────────────────────┘
```

상태 머신:
```
phase: playerTurn
  - 손패 포커스 / 카드 선택
  - Enter on card → 비용 체크
      ├ 타겟 필요: phase → awaitingInput.pickEnemy
      │   - 적 포커스 / Enter → 카드 실행
      │   - Esc → phase → playerTurn
      └ 타겟 불필요: 즉시 실행 → phase → resolving → playerTurn
  - E: 턴 종료 → phase → resolving (end-of-turn effects) → enemyTurn → playerTurn

phase: enemyTurn
  - 자동 진행 (로그 표시, 잠시 대기 후 다음)
  - 입력 무시

phase: won/lost
  - won: Reward Screen 진입
  - lost: 사망 화면 → 슬롯 wipe → TitleScreen
```

## 상태 전이 표 (요약)

| 현재 | 이벤트 | 다음 |
|---|---|---|
| Title | 슬롯 선택 (빈) | SlotMenu → NewCharacter |
| Title | 슬롯 선택 (있음) | SlotMenu (이어/새로) |
| NewCharacter | 이름 확정 | StartPhase |
| StartPhase | 구매/스킵 | RunScreen (Map) — 첫 노드는 여정의 시작 |
| Map | 노드 이동 | Event/Combat/Shop ... |
| Event | end step | Map (또는 combatStart → Combat) |
| Combat | won | RewardScreen → Map |
| Combat | lost | DeathScreen → Title |
| Map | restNode 진입 | RestHubScreen |
| RestHub | "출발" | StartPhase |
| RestHub | "타이틀" | (저장) Title |

## 페이즈와 자동 저장 매트릭스

| 페이즈 | 저장 시점 | 저장 범위 |
|---|---|---|
| Title | — | (이미 전부 저장된 상태) |
| StartPhase 진입 시 | Y | Global + Slot |
| 카드/스킬 구매 완료 후 | Y | Global + Slot |
| 노드 이동 직후 | Y | Slot.run |
| 전투 시작 직전 | Y | Slot.run |
| 전투 종료 직후 | Y | Slot.run |
| 이벤트 종료 후 | Y | Slot.run |
| 휴식처 진입 | Y | Global + Slot |
| 휴식처 내 액션 | Y (즉시) | Global + Slot |
| 사망 슬롯 wipe | Y | Global + Slot.empty |
| 종료 (Title→종료) | Y | Global + 모든 Slot |

## 입력 표준 (모든 화면 공통)

| 키 | 동작 |
|---|---|
| ↑ ↓ ← → | 포커스 이동 / 선택지 탐색 |
| Enter | 확정 |
| Esc | 취소 / 한 단계 뒤로 |
| E | (전투) 턴 종료 |
| M | (맵) 전체 보기 토글 |
| I | (탐험 중) 보유 카드 보기 (read-only) |
| ? | 도움말 모달 |
| Ctrl+S | 수동 저장 (호환성) |
| Ctrl+Q | 안전 종료 |

## 게임 종료/리스타트

- **종료 (Title→종료)**: 모든 데이터 저장 + 프로세스 종료
- **`Ctrl+C` 강제**: 마지막 저장점 손실 (직전 자동 저장은 살아있음)
- **사망**: 슬롯 wipe + Title (글로벌은 유지)

## 미정 (TBD)

- **휴식처 도중 캐릭터 강제 사망?** 없음 (휴식처는 안전 공간).
- **던전 내 휴식 노드**: 일부 디자인에서 던전 중간 휴식 노드가 있을 수 있음. v1 외 — 휴식처 = 던전 종료 노드 1개만.
- **선택지 키 단축키**: 숫자 키 (1~9) 로 직접 선택 지원할지. v1 외, 방향키만.
- **이벤트 텍스트 스킵 속도**: 한 줄씩 vs 통째 표시. 디자인 결정.
